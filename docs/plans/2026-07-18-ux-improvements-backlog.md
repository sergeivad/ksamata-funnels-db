# Бэклог доработок админки (UX-ревью 2026-07-18)

Итоги прохода по UI (список → карточка → справочники) + код-ревью.
Общий вывод: база добротная, карточка воронки удобная; ниже — что докрутить.
Приоритеты: P1 — мешает каждый день, P2 — скоро начнёт мешать, P3 — по случаю.

Обновление 2026-07-18 (вечер): P1/P2 реализованы четырьмя агентами
(поиск+фильтр, fail-closed auth, CRUD справочников, dirty-гард), пройдено
мультиагентное код-ревью (8 углов), 2 найденных бага исправлены
(immutable tags kind, product_durations → 409 вместо 500). 197/197 тестов.

## P1. Список воронок — СДЕЛАНО

## P1. Авторизация fail-closed — СДЕЛАНО

- [x] `middleware.ts`: в production без валидной `ADMIN_BASIC_AUTH` — 503 на всё;
      dev-режим без изменений. Логика вынесена в чистую `resolveAuthDecision`,
      17 тестов. `DEPLOY.md` дополнен обязательной переменной.

## P2. Справочники: редактирование — СДЕЛАНО

- [x] PATCH/DELETE `/api/refs/[kind]/[id]`: переименование обновляет зеркальные
      АВ-теги (rename-or-merge в одной транзакции), удаление только при 0
      использований (учёт и FK, и АВ-тегов), 409 с счётчиком.
- [x] UI в `RefTable`: карандаш (инлайн-редактирование, Enter/Esc) + корзина
      на hover, ошибки построчно.
- [x] Пост-ревью фиксы: kind=tags неизменяем через API и read-only в UI
      (АВ-теги управляются автоматически); удаление продукта с
      `product_durations` → 409, а не 500.
- [ ] Проверить данные: в «Подрядчиках» лежат «Органика» и «Перелив» — похоже,
      не тот справочник. Теперь можно вычистить через UI (удаление покажет,
      используются ли они).

      извлечь хук `useCopyFlash`.
- [ ] Регекс `openableUrl` и JSX внешней ссылки продублированы в UrlInput и
      BlockListField — общий помощник/компонент `OpenLinkButton`.
- [ ] Паттерн `onDirtyChangeRef` повторён в 3 компонентах — хук `useDirtyReporter`.
- [ ] `FunnelIdentity`: 9 полей идентификации перечислены трижды (snapshot,

      разъехавшийся текст тега делает значение «неиспользуемым». Симптом общей
      хрупкости денормализованных осей (см. Техдолг).

- [ ] Сборка `FunnelListItem` продублирована ~5 раз в `funnels.ts`; имя воронки
      считается в двух местах.
- [ ] Легаси-колонки в `schema.ts` (`*Raw`, `*Url`, `sheetName`…) — вычистить миграцией.
- [ ] Оси воронки восстанавливаются парсингом тегов «АВ » (`ab-tags.ts`) — хрупко;
