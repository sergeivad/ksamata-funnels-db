# Дизайн: статус «Архив» для воронок

Дата: 2026-07-19
Проект: Funnels Admin (Next.js GUI поверх БД)

## Цель

Добавить третий статус воронки — **«Архив»** — к существующим «Активна» и «Черновик».
В архив складываются уже не работающие воронки: они убираются из основного вида, но
не удаляются и остаются доступными.

## Принятые решения

- **Видимость:** вкладка «Все» показывает только рабочие воронки (активные + черновики).
  Архивные видны только на отдельной вкладке «Архив».
- **Управление:** бинарный тумблер Play/Pause в карточке заменяется на меню из трёх
  действий (Сделать активной / В черновик / В архив).
- **Без миграции БД:** колонка `status` — обычный TEXT без CHECK-ограничения, значение
  `'archive'` помещается в существующую колонку.

## 1. Модель данных

- `app/src/db/schema.ts:63` — `status: text('status').default('active')`. Тип TEXT, без
  enum/CHECK, дефолт `'active'`. **Не меняется.**
- Живая БД: колонка `status TEXT DEFAULT 'active'` без CHECK — любое строковое значение
  допустимо на уровне БД. **Миграция БД не требуется.**
- Единственный источник правды для набора статусов на уровне приложения — Zod-enum в
  `app/src/lib/validation.ts:52`: `status: z.enum(['active', 'draft'])`.
  Расширяется до `z.enum(['active', 'draft', 'archive'])`. `funnelUpdateSchema` — это
  `.partial()` от create-схемы, обновится автоматически.

## 2. Единый тип статуса (чистка)

Сейчас union `'active' | 'draft'` продублирован в:
- `app/src/components/StatusPill.tsx:2`
- `app/src/components/FunnelCard.tsx:11`
- `app/src/app/page.tsx:15` (тип фильтра `'all' | 'active' | 'draft'`)

Вводим один источник: новый модуль `app/src/lib/status.ts` с типом
`FunnelStatus = 'active' | 'draft' | 'archive'` и массивом метаданных статусов
(значение → русский лейбл). Zod-enum в `validation.ts` строится из этого же списка. Бейдж, фильтр, меню и валидация ссылаются на одно определение.

Обоснование: без этого `'archive'` пришлось бы вписывать руками в 4–5 мест; единый тип
служит непосредственно задаче и снижает риск рассинхрона.

## 3. Список и фильтр (`app/src/app/page.tsx`)

- `STATUS_OPTIONS` (`:17-21`) и тип `StatusFilter` (`:15`) расширяются: сегментированный
  фильтр становится **«Все / Активные / Черновики / Архив»**.
- Предикат фильтра (`:242`) меняется:
  - `'all'` → показывает всё **кроме** архива: `f.status !== 'archive'`.
  - `'active'` / `'draft'` → как есть (`f.status === statusFilter`).
  - `'archive'` → `f.status === 'archive'`.
- `isStatusFilter` (`:23-25`) и localStorage-ключ `LS_STATUS_KEY` (`:13`) принимают
  новое значение `'archive'`.

## 4. Бейдж статуса (`app/src/components/StatusPill.tsx`)

- Union-тип (`:2`) расширяется до трёх значений.
- Добавляется третья ветка **«Архив»** — приглушённо-серый бейдж, визуально отличимый
  от «Черновика» (черновик сейчас бежевый `bg-[#E8E4DA]` / текст `#5E5A52`). Предлагаемый
  старт: `bg-[#E0E0E0]` / текст `#6B6B6B`; финальный цвет подбирается на превью.
- Текущее поведение «любой не-active → Черновик» устраняется: явная ветка на `archive`.
- Потребители бейджа (`FunnelCard.tsx:52`, `FunnelCompactView.tsx:44`) изменений не
  требуют.

## 5. Управление статусом — меню из трёх действий

### Быстрое действие в списке
- `app/src/components/FunnelCard.tsx:62-69` — кнопка-тумблер Play/Pause заменяется на
  меню «···» с действиями: **Сделать активной / В черновик / В архив**. Текущий статус
  помечен как активный/задизейблен.
- `app/src/app/page.tsx:143` — обработчик переписывается с бинарного
  `funnel.status === 'active' ? 'draft' : 'active'` на «установить конкретный статус»
  (аргумент — целевой статус) с оптимистичным PATCH на `/api/funnels/{id}` и тостом.

### Форма редактирования воронки (`app/src/components/FunnelIdentity.tsx`)
- `:139` — в сегментированный переключатель добавляется третья опция «Архив».
- `:30` и `:42` — убирается схлопывание не-active → draft
  (`useState(funnel.status === 'active' ? 'active' : 'draft')`): значение статуса
  используется как есть из трёх допустимых.

### API
- `app/src/app/api/funnels/route.ts` (POST) и `app/src/app/api/funnels/[id]/route.ts`
  (PATCH) — изменений в коде не требуют, начинают принимать `archive` автоматически
  через обновлённую Zod-схему.

## 6. Тесты

- `app/tests/validation.test.ts` — enum теперь принимает `archive`; отклоняет мусор.
- `app/tests/api-funnels.test.ts` / `api-funnels-route.test.ts` — create/update с
  `status: 'archive'` проходит.
- Новый тест на предикат фильтра: «Все» исключает архив; вкладка «Архив» показывает
  только архивные.
- Сценарный тест: перевод воронки в архив → исчезает из «Все» → появляется во вкладке
  «Архив»; разархивация возвращает обратно.

## Вне scope (YAGNI)

- Миграция БД, CHECK-constraint на статус.
- «Мягкое удаление», авто-архивация по дате/неактивности.
- Изменение экспорта CSV — архивные строки экспортируются как есть
  (`app/src/lib/export.ts`).
- Массовые операции (архивировать несколько воронок разом).
