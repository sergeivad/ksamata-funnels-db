# Добавление АВ-тегов (новая таксономия) в базу воронок

> Дата: 2026-06-03
> Источник правил: `getcourse-taxonomy-reference.md` (АВ-таксономия GetCourse школы ksamata)

## Цель

Разметить 32 воронки в `ksamata_funnels.db` новыми тегами по 7-осевой таксономии
(`АВ ...`), **сохранив все старые (legacy) теги**. Результат должен отображаться
в выгрузке `Сводная_таблица_автоворонок.xlsx` рядом со старыми тегами.

## Принципы

- Старые теги НЕ удаляются и НЕ изменяются. Новые добавляются рядом.
- Формат тега: `АВ <Ось>: <Значение>` (пробелы вокруг `:`). Исключение —
  служебный `АВ Автоворонка` (без двоеточия), ставится всегда.
- Один Канал и один Подрядчик на воронку (текущий владелец трафика).
- «Не угадывать» — все неоднозначные воронки уточнены у пользователя (см. ниже).

## Хранение (двойное — raw + нормализованное)

1. **`tags`**: каждое новое значение тега заводится отдельной строкой
   (`АВ Автоворонка`, `АВ Этап: Регистрация`, `АВ Продукт: ДБО`, …).
2. **`funnel_tags`**: связи пишутся в существующие слоты `tag_type`:
   `reg`, `time_15`, `time_19`. Схема НЕ меняется. `position` продолжает
   нумерацию после последнего legacy-тега в слоте.
3. **raw-строки** `reg_tags_raw` / `tag_15_raw` / `tag_19_raw`: АВ-теги
   дописываются в хвост через `, ` после legacy-тегов.

## Раскладка осей по слотам

Общие оси (Продукт, Канал, Направление, Подрядчик) одинаковы во всех слотах
воронки. Различаются Этап и Время:

- **`reg`**: `АВ Автоворонка`, `АВ Этап: Регистрация`, Продукт, Канал,
  Направление, Подрядчик. (Время НЕ ставится — в legacy reg нет `t=`.)
- **`time_15`**: `АВ Автоворонка`, `АВ Этап: Оплата`, Продукт, Канал,
  Направление, Подрядчик, `АВ Время: 15`.
- **`time_19`**: то же, что time_15, но `АВ Время: 19`.

Порядок осей в строке (как в справочнике):
Автоворонка → Этап → Продукт → Канал → Направление → Подрядчик → Время.

Слот размечается только если его raw-строка непустая:
- №1 — только `reg`.
- №16, №20 — `reg` + `time_19`.
- Остальные — `reg` + `time_15` + `time_19`.

### Решения по неоднозначным воронкам (от пользователя)

- **№5**: Направление = `Реклама` (без страны).
- **№12**: Направление = `Реклама`; **№14, №15**: Направление = `Маркетплатформа`
  (новое значение оси, расшифровка legacy «МП»).
- **№16**: теперь подрядчик `ИНХАУЗ` (ранее в колонке NR / теги «ВК БАИНГ»).
- **№17**: подрядчик `NR`, Канал `МАКС`, Направление `Посевы`
  (по правилу справочника МАКС+FAQ→NR).
- **№20**: подрядчик `ИНХАУЗ`. №16 и №20 используют одни и те же вебинар-комнаты
  (`boo*-kvch`), но это разные точки входа трафика (лендинги `/nr/` и `/vkb/`).
  Пользователь планирует свести их в один ленд под ИНХАУЗ — **в рамках этой задачи
  ничего не удаляем**, обе воронки получают идентичные ИНХАУЗ-теги.

### Новые значения вне дословного справочника (заданы пользователем)

- Ось Направление: `Маркетплатформа`.
- Ось Подрядчик: `ИНХАУЗ`.

## Реализация

- Идемпотентный Python-скрипт `add_av_tags.py` (по образцу существующих
  `add_*.py`): читает маппинг, заводит недостающие теги в `tags`, добавляет
  связи в `funnel_tags` (с проверкой на дубли через `UNIQUE`), дописывает
  raw-строки. Повторный запуск не плодит дубли.
- Скрипт пишет Sonnet-агент, проверяет Opus (по договорённости).
- После запуска — пересборка `Сводная_таблица_автоворонок.xlsx` через
  существующий `ksamata_funnels_export.py`.

## Верификация

- До/после: счётчики `tags`, `funnel_tags` по `tag_type`.
- Точечная проверка 3-4 воронок (включая №1 «только reg», №16/20 ИНХАУЗ,
  №17 NR/МАКС) — что АВ-теги присутствуют и в raw, и в funnel_tags, а
  старые теги на месте.
- Идемпотентность: второй прогон не меняет счётчики.
- xlsx содержит АВ-теги рядом со старыми.
